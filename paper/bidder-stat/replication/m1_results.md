# M1 — cycle-walking decision rule

Generated by `m1_cycle_walking.py`. Backend selection in
`coupler.py:Bidder.__init__`: speck32 when
`2^32 <= MAX_CYCLE_WALK_RATIO · P` (i.e., `P >= 2^26 = 67,108,864`), feistel otherwise.

Expected calls-per-output for cycle-walking is `2^32 / P`.
`MAX_CYCLE_WALK_RATIO = 64` caps this; above the cap,
feistel takes over.

## Reading

- The crossover at `P = 2^26` is visible: the row at
  `P = 2^25` uses feistel; the row at `P = 2^26` uses
  speck32. The expected-ratio column quantifies why:
  speck32 cycle-walking would need `2^32 / 2^25 = 128`
  calls per output below the threshold, exceeding the
  `MAX_CYCLE_WALK_RATIO = 64` budget.

- At `P` close to `2^32`, the expected ratio approaches
  1 and speck32 throughput is best.

- The table's `ns / call` column is the load-bearing
  number for §4.3: it shows that the implementation's
  fixed threshold at `P = 2^26` is consistent with the
  measured throughput crossover.
