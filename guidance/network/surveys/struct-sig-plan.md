**Writing-discipline plan for a synthesis note.** The doc is
a review copy — a plan for how
`experiments/acm-champernowne/base2/STRUCTURAL-SIGNATURES.md`
should be written, not the synthesis itself. Its value as a
survey entry is that it prescribes the *seven-field shape*
that scales the chimney-climbing principle from a two-way
triad to a seven-way brace. Every observable in the binary
subtree gets seven independent framings, and the claim is
only trusted when the seven agree.

**The seven fields per observable.** (1) *Claim* — the
compact statement. (2) *Evidence in repo* — the specific
artifacts that witness it. (3) *Relation to other
observables* — how it overlaps with or is distinct from
neighbors. (4) *Statistical confidence* — how strong the
measurement is, on what sample size, against what control.
(5) *Semantic confidence* — whether we understand what the
observable is measuring in the first place. (6)
*Falsification* — what would force retraction. (7) *Next
check* — a script-runnable extension, not "investigate
further."

**Source precedence rules.** When source notes disagree
(and they do — see
[`detrended-rds.md`](detrended-rds.md)'s correction of
[`hamming-bookkeeping.md`](hamming-bookkeeping.md)), the
rules are explicit: more specific experiment note beats
older umbrella memo; later correction beats earlier
summary; closed-form derivation beats impressionistic
README language; a result with an explicit control beats a
result without one. Applied to the binary subtree, these
rules make the synthesis packet's voice consistent even
when its inputs aren't.

**Chimney-climbing role.** The doc is the Discipline
chapter's related note because it scales the
*four-triad* principle of the chapter (BQN / English /
Python, C / Python / English, Human / cold-agent /
in-session-agent, Theorem / test / experiment) into a
*seven-channel* discipline applied at the level of a single
observable. A chimney with seven walls is still a chimney:
bracing, not audit.
